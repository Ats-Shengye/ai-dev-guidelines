# AI Development Guidelines

AIコーディングアシスタント（Claude等）との協働開発を想定したガイドラインフレームワーク。

## 概要

セキュリティを重視したAI協働開発のための**骨格フレームワーク**。コーディング規約、ワークフロー定義、品質ゲートを提供する。

本リポジトリは参考用のフレームワークであり、そのまま使用可能な完全なソリューションではない。各プロジェクトの要件に応じたカスタマイズを前提とする。

## ファイル構成

| ファイル | 内容 |
|----------|------|
| **Coding.md** | コーディング規約、スタイルガイド、アーキテクチャ原則、提出前チェックリスト |
| **Workflow.md** | AI協働開発ワークフロー、エージェントチーム構成、フェーズ定義 |
| **UpdateGuide.md** | ガイドライン更新手順、レビュープロセス |

## Security.mdについて

**Security.md**は意図的に含めていない。セキュリティ設定は以下の要因によりプロジェクトごとに大きく異なるため：

- プロジェクトタイプ（Web、API、CLI、ライブラリ）
- デプロイ環境（ローカル、クラウド、オンプレミス）
- コンプライアンス要件（GDPR、HIPAA、PCI-DSS）
- リスク許容度

Coding.mdおよびWorkflow.mdはSecurity.mdをプレースホルダとして参照しており、各プロジェクトで独自のSecurity.mdを作成することを推奨する。

## 基本方針

### セキュリティ最優先
- プロジェクト規模に関わらずセキュリティ基準に妥協しない
- 適用可能なセキュリティ対策は全て実施
- 判断に迷う場合はセキュリティ対策を実施

### AI協働
- AI生成コードは必ずレビューを実施
- アーキテクチャは人間が決定し、AIは実装を支援
- 生成プロンプトを記録し再現性を確保

### 品質ゲート
- コミット前の静的解析を必須とする（mypy, ruff, bandit）
- 変更ごとに提出前チェックリストを確認
- 重要なコードパスにはセキュリティレビューを実施

## ワークフロー概要

```
ユーザー要件
     ↓
[Phase 1] Claude: 要件定義
     ↓
[Phase 1.5] Coder: 見積もり
     ↓
[Phase 2] Coder: 実装 (Coding.md準拠)
     ↓
[Phase 3] Security Reviewer: 検証 (独自Security.md準拠)
     ↓
[Phase 3.5] Claude: コード品質レビュー
     ↓
[Phase 4] Claude: 統合報告
     ↓
[Phase 5] ユーザー: 承認 or 修正指示
     ↓
   完了
```

## 使用方法

### AIアシスタント向け
システムプロンプトまたはCLAUDE.mdにて本ドキュメントを参照する。エージェントコマンド（`/coder`、`/security-review`）は概念的なプロンプト例として記載。

### 開発者向け
- コード提出前にPre-Submission Checklistを使用
- Universal Coding Principles（10項目）に準拠
- プロジェクト固有のSecurity.mdを作成

### チーム向け
- ワークフローフェーズを開発プロセスに適用
- エージェント役割（Coder、Security Reviewer）をチームメンバーまたはAIに割り当て
- 見積もり・レビュー基準をプロジェクトに合わせて調整

## カスタマイズ手順

1. 本リポジトリをフォーク
2. プロジェクト用Security.mdを作成（セキュリティ原則、レビューチェックリスト、インシデント対応を含む）
3. Coding.mdを使用言語・フレームワークに合わせて調整
4. Workflow.mdのフェーズをチーム構成に合わせて修正

## Known Limitations

Coding.mdおよびWorkflow.mdの「Known Limitations」セクションに既知の制限・改善候補を記載。
